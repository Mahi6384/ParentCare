Full Product Plan · 2026

ParentCare

Kids assign tasks. Parents complete them.
AI verifies the proof.

Next.js 14 Claude Vision BullMQ + Redis Supabase WhatsApp API WebSockets

The Problem

👥

Millions of kids live away

Students and professionals relocate for work and education — leaving parents behind with no daily check-in mechanism.

🛡️

Parents skip healthy habits

Without gentle pressure, parents skip walks, eat poorly, skip medicine. No one is watching. No one is nudging.

📱

No real accountability

Phone calls are forgotten. Existing apps are too complex for parents. There is no way to verify anything was actually done.

The Solution

A two-sided accountability platform with AI at the core.

⌨️
Kid creates task

Sets schedule, proof type & personal note

🔔
Parent gets reminder

WhatsApp deep-link + email at scheduled time

📷
Parent submits photo

One tap from WhatsApp, compressed upload

🧠
AI verifies photo

Claude Vision checks scene, food, clothing, pill label

✅
Kid gets result

Real-time WebSocket push + streak update

The AI verification layer is the differentiator — parents can't just tick a checkbox. They submit real photo proof, which AI checks for scene, food quality, and activity. Result includes a confidence score and plain-English reason.

Key Features

</>

Smart Task Builder

Templates for walks, diet, medicine, sleep, doctor visits. Kids set schedule, proof type, recurrence, and leave a personal voice note.

🧠

AI Photo Verification

Claude Vision checks scene, food quality, clothing. Returns pass / flag / fail with plain-English reason and confidence score.

💬

Multi-Channel Alerts

WhatsApp deep-links (Twilio) + Email (Resend) + in-app WebSocket bell. Parent sets their preferred channel.

🥗

Nutrition Checker

For meal tasks, Claude estimates meal name, protein level, vegetables, fruits, and flags concerns like "no greens visible".

💊

Medication Label Check

Claude reads the pill bottle label and confirms the correct medication matches what the kid specified. Prevents fake submissions.

⚡

Async Job Queue

BullMQ + Redis processes photos in the background. Prevents blocking, supports retry logic and back-pressure. Parent doesn't wait.

📈

Streaks & Dashboard

Daily streaks, 1-free-skip-per-week freeze, milestone badges. Kid sees completion rates, history, nutrition trends.

🎙️

Voice Notes on Tasks

Kid records a ≤30s voice note when creating a task. Parent hears it before submitting proof. Emotional connection built-in.

🛡️

Rate Limiting + Audit Log

Redis-based per-user rate limiting prevents abuse. Full audit log of every AI decision for transparency and trust.

AI Verification Layer

Claude claude-sonnet-4-6 Vision is the load-bearing layer of ParentCare. It's not decoration — every submission is verified before a streak is granted.

🚶 Walk Task

Checks: outdoor scene, shoes visible, natural lighting, movement context

🍽️ Meal Task

Checks: food quality, identifies meal, estimates nutrition, flags concerns

💊 Medicine Task

Reads pill bottle label, confirms correct medication name matches specification

😴 Sleep Task

Checks: dark room, person in bed, reasonable time context

Multi-Channel Alert System

💬

WhatsApp (Primary)

Twilio WhatsApp API sends a deep-link message at scheduled time. One tap opens the task directly. Works in sandbox in 10 minutes. ~₹0.4 per message.

📧

Email (Fallback)

Resend sends a styled HTML email with task details + CTA button. Also delivers the weekly PDF report every Sunday night. Free tier: 3,000/month.

🔔

In-App Bell (Real-time)

WebSocket pushes result to kid the moment Claude finishes. No refresh needed. Kid also sees when parent opens or skips a task.

System Architecture

Kid App (Next.js PWA)
  • Task creation UI with templates
  • Dashboard + streaks + badges
  • Real-time WebSocket result feed
  • Nutrition trend charts
  • Manual review panel (AI override)
  • Voice note recording
  • Family invite flow
Backend (Next.js API Routes)
  • Supabase Auth (JWT, role claims)
  • BullMQ job queue (photo verify)
  • Redis rate limiter (per-user)
  • Supabase DB + Storage
  • Claude API caller (structured JSON)
  • Twilio WhatsApp sender
  • Resend email sender
  • WebSocket server (Socket.io)
  • Cron: task scheduler + PDF report
  • Audit log writer
Parent App (Next.js PWA)
  • Today's tasks view
  • Photo upload (compressed)
  • WhatsApp deep-link entry
  • Voice note playback
  • Streak display
  • Nutrition card per meal
  • Channel preference settings
Key design decision: Photo verification is async. Parent submits → job queued in BullMQ → Claude processes in background → WebSocket pushes result to kid. Parent is never blocked waiting for AI. Retry logic (3× exponential backoff) handles Claude API failures silently.

Database Schema

users

  • id uuid PK
  • role kid | parent
  • name, phone, email
  • whatsapp_number
  • alert_preference

families

  • id uuid PK
  • kid_id → users
  • parent_id → users
  • invite_code unique
  • linked_at timestamp

tasks

  • id uuid PK
  • kid_id → users
  • title, type, template
  • schedule, recurrence
  • proof_type, note
  • voice_note_url, streak_goal

task_instances

  • id uuid PK
  • task_id → tasks
  • parent_id → users
  • due_at timestamp
  • status pending|submitted
    |passed|flagged|failed|skipped

submissions

  • id uuid PK
  • task_instance_id FK
  • photo_url (Storage)
  • submitted_at timestamp

ai_results

  • id uuid PK
  • submission_id FK
  • result, confidence
  • reason text
  • nutrition_json jsonb
  • medication_json jsonb

streaks

  • task_id, parent_id PK
  • current_streak int
  • longest_streak int
  • freeze_used_this_week bool

notifications

  • id uuid PK
  • user_id → users
  • channel whatsapp|email|ws
  • sent_at, status

audit_log

  • id uuid PK
  • submission_id FK
  • claude_prompt text
  • claude_response jsonb
  • created_at timestamp
